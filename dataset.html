cogitarelink.dataset Module

The cogitarelink.dataset module provides tools for working with structured datasets in JSON-LD format. It extends the core functionality of the LinkedDataKnowledge class with dataset-specific capabilities, enabling users to explore, query, and analyze JSON-LD datasets effectively.

Key Features: - Dataset Loading: Load JSON-LD datasets from files, URLs, or dictionaries - Structure Exploration: Navigate through complex nested structures of datasets - Path-based Analysis: Query specific paths within datasets to extract targeted information - Evidence Collection: Gather supporting evidence from datasets for specific topics or questions - Dataset Summarization: Generate concise summaries of dataset content and structure - Relationship Navigation: Follow connections between entities in the dataset - Format Conversion: Transform between different dataset representations and formats

Integration with Agents

The module includes tools that can be integrated with AI agents to enable:

  • Systematic exploration of dataset structures
  • Evidence-based analysis of dataset content
  • Question answering based on dataset information
  • Context-aware navigation through complex data relationships This module is particularly useful for working with structured data in domains like research data management, knowledge graphs, and semantic data catalogs.

Dataset Module Documentation

Here’s comprehensive documentation for the dataset module in cogitarelink:

Overview

The dataset module provides methods for manipulating and exploring JSON-LD datasets. It focuses on advanced JSON-LD 1.1 features like containers and included entities, allowing for sophisticated data organization and navigation.

Key Features

  • Structure datasets using JSON-LD 1.1 container types
  • Organize data around main entities with related entities
  • Explore datasets by path
  • Search datasets for specific information

Structure with Containers

@patch
def structure_with_containers(self:LinkedDataKnowledge,
                             property_name:str, # Property to structure
                             container_type:str, # Type: "set", "list", "language", "id", "type"
                             items:List=None, # Items to include, or None to use existing
                             base_uri:str="https://example.org/" # Base URI for properties
                             ) -> 'LinkedDataKnowledge':
    """Structure a property using JSON-LD 1.1 container features.
    
    This method allows you to create specialized collections using JSON-LD 1.1 
    container types, providing more semantic structure to your data.
    
    Args:
        property_name: Name of the property to structure
        container_type: Type of container to use
            - "set": Unordered collection of values
            - "list": Ordered collection with preserved order
            - "language": Collection of language-tagged strings
            - "id": Collection indexed by ID
            - "type": Collection indexed by type
        items: Items to include in the container (optional)
        base_uri: Base URI for the property
    
    Returns:
        self: For method chaining
    
    Examples:
        # Create a list with preserved order
        >>> kb.structure_with_containers("steps", "list", [
        ...     "Extract data", 
        ...     "Transform values", 
        ...     "Load results"
        ... ])
        
        # Create a language map
        >>> kb.structure_with_containers("label", "language", [
        ...     {"lang": "en", "value": "Person"},
        ...     {"lang": "es", "value": "Persona"},
        ...     {"lang": "fr", "value": "Personne"}
        ... ])
        
        # Create an ID-based index
        >>> kb.structure_with_containers("people", "id", [
        ...     {"id": "person1", "value": {"name": "Alice"}},
        ...     {"id": "person2", "value": {"name": "Bob"}}
        ... ])
    """

Use Included Pattern

@patch
def use_included(self:LinkedDataKnowledge,
                main_entity:Dict, # Main entity to focus on
                related_entities:List[Dict], # Related entities to include
                preserve_structure:bool=False # Whether to preserve existing structure
                ) -> 'LinkedDataKnowledge':
    """Structure knowledge using @included for related entities.
    
    This method implements the JSON-LD 1.1 pattern of organizing data around a main
    entity with related entities in an @included array. This provides a more
    resource-centric view of the data.
    
    Args:
        main_entity: The main entity to focus on
        related_entities: List of related entities to include
        preserve_structure: If True, preserves existing structure and adds @included;
                           if False, replaces structure with resource-centric view
    
    Returns:
        self: For method chaining
    
    Examples:
        # Simple case - replace structure with resource-centric view
        >>> kb.use_included(
        ...     main_entity={"@id": "ex:person1", "@type": "Person", "name": "Alice"},
        ...     related_entities=[
        ...         {"@id": "ex:org1", "@type": "Organization", "name": "Acme Inc"}
        ...     ]
        ... )
        
        # Preserve existing structure
        >>> kb.use_included(
        ...     main_entity={"@id": "ex:person1", "@type": "Person", "name": "Alice"},
        ...     related_entities=[
        ...         {"@id": "ex:org1", "@type": "Organization", "name": "Acme Inc"}
        ...     ],
        ...     preserve_structure=True
        ... )
    """

Working with Datasets

Loading Datasets

Datasets can be loaded directly into the LinkedDataKnowledge class:

# Load from a dictionary
kb = LinkedDataKnowledge(my_dataset)

# Load from a JSON file
import json
with open('dataset.json', 'r') as f:
    kb = LinkedDataKnowledge(json.load(f))

Exploring Datasets

Use the describe method to explore dataset structure:

# Describe the root structure
kb.describe()

# Describe a specific path
kb.describe("recordSet[0].field")

Searching Datasets

Use the search_dataset tool to find specific information:

# Search for a term
results = search_dataset("Person")

# Case-sensitive search
results = search_dataset("recordSet", case_sensitive=True)

Advanced Usage Examples

Creating a Dataset with Multiple Container Types

# Start with a basic knowledge base
kb = LinkedDataKnowledge()

# Add a list of steps
kb.structure_with_containers("steps", "list", [
    "Extract data", 
    "Transform values", 
    "Load results"
])

# Add language-specific labels
kb.structure_with_containers("label", "language", [
    {"lang": "en", "value": "Data Pipeline"},
    {"lang": "es", "value": "Tubería de Datos"},
    {"lang": "fr", "value": "Pipeline de Données"}
])

# Add type-based entities
kb.structure_with_containers("entities", "type", [
    {"type": "Person", "value": {"name": "Alice"}},
    {"type": "Organization", "value": {"name": "Acme Inc"}}
])

Organizing Data with Main and Related Entities

# Create a resource-centric view
kb.use_included(
    main_entity={
        "@id": "ex:dataset1",
        "@type": "Dataset",
        "name": "Sample Dataset",
        "description": "A sample dataset for demonstration",
        "creator": {"@id": "ex:person1"}
    },
    related_entities=[
        {
            "@id": "ex:person1",
            "@type": "Person",
            "name": "Alice Smith",
            "email": "alice@example.com"
        },
        {
            "@id": "ex:org1",
            "@type": "Organization",
            "name": "Example Corp",
            "url": "https://example.com"
        }
    ]
)

Best Practices

  1. Use containers appropriately:
    • Use @list when order matters
    • Use @language for multilingual content
    • Use @id for entity references
    • Use @type for type-based collections
  2. Structure related entities with @included:
    • Put the main focus entity at the root
    • Include related entities in @included
    • Use @id references to link entities
  3. Explore gradually:
    • Start with the root structure
    • Explore specific paths as needed
    • Use search to find relevant information
  4. Preserve context:
    • Keep the @context section consistent
    • Use the same vocabulary terms throughout
