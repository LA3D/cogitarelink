The vocabulary module has been enhanced to support dynamic knowledge acquisition from Wikidata, one of the world’s largest structured knowledge bases. This enables LLM agents to expand their knowledge on demand, following the “follow your nose” principle of Linked Data.

Vocabulary Module: Dynamic Knowledge Acquisition

Overview

The vocabulary module provides functionality for fetching, parsing, and manipulating RDF vocabularies in various formats. It enables the retrieval of vocabularies from remote sources and their conversion into a consistent JSON-LD representation for use within the cogitarelink system.

Key Features

  • Content negotiation for different RDF formats (JSON-LD, RDF/XML, Turtle, etc.)
  • Extraction of embedded JSON-LD from HTML pages
  • Special handling for common vocabulary sources (Schema.org, etc.)
  • Support for JSON-LD 1.1 features like scoped contexts
  • Rich visualization and summary capabilities

Key Enhancements

  1. Wikidata Search: Find entities and properties in Wikidata using natural language queries
  2. Entity Fetching: Retrieve detailed information about specific entities
  3. Named Graph Storage: Store Wikidata entities as separate named graphs in the agent’s memory
  4. Integration with Core: Seamless integration with the core module’s memory management features

Agent Workflow

These enhancements enable a powerful workflow for LLM agents:

  1. Discovery: Search Wikidata for entities related to a topic
  2. Selection: Choose the most relevant entity based on descriptions
  3. Acquisition: Fetch and store the selected entity as a named graph
  4. Exploration: Navigate the entity’s properties and relationships
  5. Integration: Connect the new knowledge with existing knowledge

This workflow allows agents to dynamically expand their knowledge base as needed, rather than relying solely on pre-loaded information. Copy

Error Handling

The module includes robust error handling for various scenarios:

  • Network errors when fetching vocabularies
  • Invalid or unsupported RDF formats
  • Malformed JSON-LD in HTML pages
  • Missing or inaccessible vocabularies

Errors are reported with descriptive messages to help diagnose and resolve issues.

Best Practices

  1. Enable debug mode when fetching new vocabularies to see what’s happening behind the scenes
  2. Use content negotiation by default to get the best available format
  3. Create scoped contexts to organize domain-specific knowledge
  4. Summarize vocabularies to understand their structure before using them
  5. Handle errors gracefully as vocabulary sources may change or become unavailable

Why Vocabularies Matter for AI Agents

Vocabularies (also called ontologies or schemas) provide the semantic foundation for linked data. They define:

  1. Terms - Concepts and properties with unique identifiers
  2. Relationships - How terms relate to each other
  3. Constraints - Rules about how terms can be used
  4. Semantics - Precise meaning of concepts

For AI agents to effectively work with linked data, they must be able to: - Retrieve vocabulary definitions - Understand the meaning of terms - Navigate relationships between concepts - Apply vocabulary knowledge to interpret data

This module gives agents these capabilities by providing tools to fetch, explore and traverse vocabularies in JSON-LD format.

Vocabulary Graph Structure

The cogitarelink framework organizes fetched vocabularies and entities into a consistent graph structure, making it easy for LLM agents to navigate and use the knowledge.

Graph ID Patterns

All vocabulary graphs follow a consistent ID pattern:

  1. Wikidata Entities: vocabulary:wikidata:entity:{QID}
    • Example: vocabulary:wikidata:entity:Q42 (Douglas Adams)
    • Consistent regardless of how the entity was referenced (QID, URL, prefixed ID)
  2. Schema.org: vocabulary:schema.org
    • Consistent single graph for all Schema.org vocabulary
  3. Standard Vocabularies: vocabulary:{domain}:{path}
    • Example: vocabulary:purl.org:dc:terms (Dublin Core Terms)
    • Example: vocabulary:www.w3.org:2004:02:skos:core (SKOS)

Graph Metadata

Each vocabulary graph includes consistent metadata:

  • title: Human-readable name of the vocabulary
  • source: Original URI from which the vocabulary was fetched
  • vocabulary: Base URI of the vocabulary
  • entityCount: Number of entities in the graph
  • lastUpdated: Timestamp of when the graph was fetched

Benefits for LLM Agents

This consistent structure enables agents to:

  1. Locate Knowledge: Find entities across different vocabularies
  2. Understand Sources: Identify where knowledge came from
  3. Navigate Relationships: Follow links between entities in different graphs
  4. Update Knowledge: Refresh specific vocabularies as needed

Agents can use the find_entity_across_graphs method to search for entities regardless of which vocabulary they come from.

        if graph_id == "main" or full_graph_id == "did:cogitarelink:graph:main":

        else:

    if entity_id:
        if debug: print(f"Looking for entity: {entity_id}")
        # Find the entity in the graph
        entity = None
        
        # Look in @graph
        for e in graph_data.get('@graph', []):
            if e.get('@id') == entity_id or e.get('@id').endswith(entity_id):
                entity = e
                if debug: print(f"Found entity in @graph: {e.get('@id')}")
                break

        if not entity and '@included' in graph_data:
            for e in graph_data['@included']:
                if e.get('@id') == entity_id or e.get('@id').endswith(entity_id):
                    entity = e
                    if debug: print(f"Found entity in @included: {e.get('@id')}")
                    break
        
        # If not found, check if the main entity itself matches
        if not entity and '@id' in graph_data and (graph_data['@id'] == entity_id or graph_data['@id'].endswith(entity_id)):
            entity = graph_data
            if debug: print(f"Found as main entity: {graph_data['@id']}")
        
        if entity:
            # If a specific property is requested
            if property_name:
                if debug: print(f"Looking for property: {property_name}")
                
                # Handle both direct property and prefixed versions
                matching_props = [p for p in entity.keys() if 
                                 p == property_name or 
                                 p.endswith(f":{property_name}") or 
                                 p.endswith(f"/{property_name}") or 
                                 p.endswith(f"#{property_name}")]
                
                if matching_props:
                    prop = matching_props[0]
                    if debug: print(f"Found property: {prop}")
                    return f"# Property: {prop}\n\n```json\n{json.dumps(entity[prop], indent=2)}\n```"
                else:
                    if debug: print(f"Property not found: {property_name}")
                    return f"Property not found: {property_name}"
            
            # Return the full entity
            return f"# Entity: {entity.get('@id')}\n\n```json\n{json.dumps(entity, indent=2)}\n```"
        
        if debug: print(f"Entity not found: {entity_id}")
        return f"Entity not found: {entity_id}"
    
    # Otherwise, provide an overview of the graph
    entities = graph_data.get('@graph', [])
    
    # Also include entities from @included if present
    included_entities = graph_data.get('@included', [])
    
    # Include the main entity if it has an @id
    main_entity = None
    if '@id' in graph_data:
        main_entity = {k: v for k, v in graph_data.items() if k not in ['@graph', '@included', '@context']}
        if '@id' in main_entity:

            if debug: print(f"Including main entity: {main_entity.get('@id')}")

        if metadata:

                    metadata_str += f"- **{key}**: {value}\n"

                        break
            
            if main_entity_label:
                break
    
    # Add the label to the output if found
    if main_entity_label:
        output.insert(2, f"**Main entity label**: {main_entity_label}")
    
    # Add metadata
    output.append(metadata_str)
    
    # Count entity types across both @graph and @included
    type_counts = {}
    for entity in all_entities:
        entity_type = entity.get('@type')
        if isinstance(entity_type, list):
            for t in entity_type:
                type_counts[t] = type_counts.get(t, 0) + 1
        elif entity_type:
            type_counts[entity_type] = type_counts.get(entity_type, 0) + 1
    
    if type_counts:
        output.append("## Entity Types")
        for t, count in sorted(type_counts.items(), key=lambda x: x[1], reverse=True)[:10]:
            output.append(f"- {t}: {count}")
        if len(type_counts) > 10:
            output.append(f"- ... and {len(type_counts) - 10} more types")

    # Sample entities from both @graph and @included
    # Try to prioritize entities with labels
    labeled_entities = []
    unlabeled_entities = []
    
    for entity in all_entities:
        has_label = False
        for key in entity.keys():
            if 'label' in key.lower() or key == 'name':
                has_label = True
                break
        
        if has_label:
            labeled_entities.append(entity)
        else:
            unlabeled_entities.append(entity)
    
    # Combine, prioritizing labeled entities
    sample_entities = (labeled_entities + unlabeled_entities)[:sample_size]
    
    output.append(f"## Sample Entities (showing {len(sample_entities)} of {total_entities})")
    for entity in sample_entities:
        entity_id = entity.get('@id', 'Unknown ID')
        entity_type = entity.get('@type', 'Unknown Type')

            entity_type = ', '.join(entity_type)

        label = None
        
        # First check for Wikidata simplified label
        if 'label' in entity and isinstance(entity['label'], str):
            label = entity['label']
        else:
            # Check various label properties
            for key, value in entity.items():
                if 'label' in key.lower() or key == 'name':
                    if isinstance(value, list):
                        for item in value:
                            if isinstance(item, dict) and '@value' in item:
                                label = item['@value']
                                break
                            elif isinstance(item, str):
                                label = item
                                break
                    elif isinstance(value, str):
                        label = value
                    break
        
        if label:
            output.append(f"- **{entity_id}** (Type: {entity_type}) - *{label}*")
        else:
            output.append(f"- **{entity_id}** (Type: {entity_type})")
    
    return "\n".join(output)
